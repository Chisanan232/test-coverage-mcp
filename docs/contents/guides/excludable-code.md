# Excludable Code Detection and Conservative Approach

## Overview

The Excludable Code Candidate Service identifies code that should be excluded from coverage metrics. It uses a conservative approach to minimize false positives and ensure only truly excludable code is flagged.

## Detection Signals

### 1. Generated Code

**Indicators**:
- Explicit markers: `AUTO-GENERATED`, `DO NOT EDIT`, `GENERATED CODE`
- File extensions: `.pb2.py`, `.pb2_grpc.py`, `.g.dart`
- Generation patterns: "generated by", "generated from", "code generator"

**Confidence Scoring**:
- Explicit marker: +0.3
- Generation pattern: +0.2
- File extension: +0.4

**Threshold**: > 0.5 confidence

**Example**:
```python
# AUTO-GENERATED
# DO NOT EDIT
# This file is generated by protoc

class GeneratedMessage:
    pass
```

### 2. Vendor Code

**Indicators**:
- Vendor directories: `vendor/`, `node_modules/`, `.venv/`, `site-packages/`
- Third-party paths: `third_party/`, `external/`
- Common vendor files: `jquery.js`, `bootstrap.js`, `react.js`

**Confidence Scoring**:
- Vendor directory: +0.4
- Vendor file: +0.5

**Threshold**: > 0.5 confidence

**Example**:
```
node_modules/package/index.js
.venv/lib/python/site-packages/module.py
vendor/lib/module.py
```

### 3. Migration/Deprecated Code

**Indicators**:
- Deprecation markers: `DEPRECATED`, `LEGACY`
- Migration comments: "will be removed", "to be removed", "old implementation"
- Version-specific paths: `v1/`, `v2/`

**Confidence Scoring**:
- Deprecation marker: +0.25
- Migration comment: +0.2
- Version-specific path: +0.15

**Threshold**: > 0.5 confidence

**Example**:
```python
# DEPRECATED - Use new_function instead
# TODO: remove in v2.0
def old_function():
    pass
```

### 4. Zero-Hit Signals

**Indicators**:
- File has zero hits
- All lines uncovered
- Unreachable code patterns: `if False:`, `raise NotImplementedError`
- Not-implemented code

**Confidence Scoring**:
- Zero hits: +0.3
- All uncovered: +0.3
- Unreachable pattern: +0.4
- Not-implemented: +0.2

**Threshold**: > 0.5 confidence

**Example**:
```python
if False:
    def unreachable():
        pass

raise NotImplementedError("Not yet implemented")
```

## Conservative Detection Approach

The service uses strict criteria to avoid false positives:

### Disqualifying Factors

Code is NOT excluded if it:

1. **Defines Public API**
   - Has `__all__` definition
   - Contains public classes/functions

2. **Has Side Effects**
   - Modifies `sys.path`
   - Accesses `os.environ`
   - Performs monkey patching
   - Modifies global state

3. **Is Test Code**
   - Located in test directories
   - Named with test patterns

### Confidence Thresholds

| Confidence | Action |
|-----------|--------|
| < 0.3 | Insufficient evidence |
| 0.3-0.5 | Weak signal, needs review |
| 0.5-0.7 | Moderate confidence |
| 0.7-0.9 | High confidence |
| > 0.9 | Very high confidence |

## Excludability Scoring

The service calculates weighted excludability score:

```
excludability_score = (
    generated_confidence * 0.4 +      # 40% weight
    vendor_confidence * 0.3 +         # 30% weight
    migration_confidence * 0.2 +      # 20% weight
    zero_hit_confidence * 0.1         # 10% weight
)
```

**Decision**:
- Score > 0.5: Should exclude
- Score ≤ 0.5: Should not exclude

## Examples

### Example 1: Generated Code

```python
# file.pb2.py
# AUTO-GENERATED
# DO NOT EDIT

class GeneratedMessage:
    def __init__(self):
        pass
```

**Analysis**:
- Explicit marker: +0.3
- File extension `.pb2.py`: +0.4
- Total confidence: 0.7 (> 0.5) → **GENERATED**

**Recommendation**: Safe to exclude

### Example 2: Vendor Code

```python
# node_modules/package/index.js
(function() {
    // Third-party library code
})();
```

**Analysis**:
- Vendor directory `node_modules/`: +0.4
- Total confidence: 0.4 (≤ 0.5) → **NOT VENDOR** (borderline)

**Recommendation**: Review manually

### Example 3: Deprecated Code

```python
# src/legacy.py
# DEPRECATED - Use new_module instead
# TODO: remove in v3.0

def old_function():
    """Old implementation."""
    pass
```

**Analysis**:
- Deprecation marker: +0.25
- Migration comment: +0.2
- Total confidence: 0.45 (≤ 0.5) → **NOT MIGRATION** (borderline)

**Recommendation**: Review manually, consider removing

### Example 4: Code with Public API

```python
# src/generated.py
# AUTO-GENERATED

__all__ = ['PublicClass']

class PublicClass:
    def public_method(self):
        pass
```

**Analysis**:
- Generated marker: +0.3
- Has `__all__`: **DISQUALIFIED**

**Recommendation**: Cannot exclude (has public API)

### Example 5: Code with Side Effects

```python
# src/config.py
# AUTO-GENERATED

import sys
sys.path.insert(0, '/custom/path')

class Config:
    pass
```

**Analysis**:
- Generated marker: +0.3
- Modifies `sys.path`: **DISQUALIFIED**

**Recommendation**: Cannot exclude (has side effects)

## Best Practices

### 1. Use Explicit Markers

Mark generated code clearly:

```python
# AUTO-GENERATED
# DO NOT EDIT
# Generated by: protoc --python_out=.
```

### 2. Organize Vendor Code

Keep vendor code in separate directories:

```
project/
├── src/
│   ├── main.py
│   └── utils.py
├── vendor/
│   └── third_party_lib/
└── node_modules/
```

### 3. Mark Deprecated Code

Use clear deprecation markers:

```python
# DEPRECATED - Use new_function instead
# TODO: remove in v2.0
def old_function():
    """Old implementation."""
    pass
```

### 4. Avoid False Positives

Don't mark code as excludable if it:
- Exports public APIs
- Has side effects
- Is actively maintained
- Has business logic

## Integration with Configuration

### Codecov Configuration

```yaml
coverage:
  precision: 2
  round: down
  range: "70...100"

ignore:
  - "vendor"
  - "node_modules"
  - "**/__pycache__"
  - "**/test/**"
  - "**/*.pb2.py"
```

### Coverage.py Configuration

```ini
[run]
omit =
    */tests/*
    */test_*.py
    */__pycache__/*
    */site-packages/*
    */vendor/*
    *.pb2.py
```

## Validation Checklist

Before excluding code, verify:

- [ ] Code is definitely generated/vendor/deprecated
- [ ] Code doesn't define public APIs
- [ ] Code has no side effects
- [ ] Code is not actively maintained
- [ ] Exclusion won't hide real issues
- [ ] Team agrees on exclusion

## Limitations

1. **Pattern Matching**: Uses simple patterns, not AST analysis
2. **False Negatives**: May miss some excludable code
3. **False Positives**: May incorrectly flag code as excludable
4. **Context Blind**: Doesn't understand business logic
5. **Manual Review**: Recommendations should be reviewed

## Common Mistakes

### Mistake 1: Excluding Code with Public API

```python
# ❌ WRONG - Has public API
# AUTO-GENERATED
__all__ = ['PublicClass']
class PublicClass:
    pass
```

### Mistake 2: Excluding Code with Side Effects

```python
# ❌ WRONG - Has side effects
# AUTO-GENERATED
import sys
sys.path.insert(0, '/path')
```

### Mistake 3: Excluding Active Code

```python
# ❌ WRONG - Still actively used
# DEPRECATED
def important_function():
    pass
```

### Mistake 4: Over-Excluding

```python
# ❌ WRONG - Too broad
exclude_paths = ["src/**"]  # Excludes everything!
```
