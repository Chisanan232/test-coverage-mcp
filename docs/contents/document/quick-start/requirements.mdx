# Requirements

Before using test-coverage-mcp, ensure your development environment meets the following requirements.

## System Requirements

- **Operating System**: Linux, macOS, or Windows
- **Memory**: Minimum 512MB RAM (1GB+ recommended for development)
- **Disk Space**: Minimum 200MB for installation and dependencies
- **Internet Connection**: Required for downloading dependencies and accessing coverage providers

## Core Prerequisites

### Python Version

- **Python**: Version 3.12 or higher (up to 3.14 supported)
- **Recommended**: Python 3.13 for optimal compatibility

:::tip Python Installation
If you don't have Python installed:
- **macOS**: Use `brew install python` or download from [python.org](https://www.python.org/)
- **Linux**: Use your package manager (e.g., `apt install python3.13`)
- **Windows**: Download from [python.org](https://www.python.org/) or use `winget install Python.Python.3.13`
:::

### Package Manager

Choose one of the following:

- **pip** (included with Python): Standard Python package manager
- **uv** (recommended): Fast, modern Python package manager
  - Install: `pip install uv` or see [uv installation guide](https://docs.astral.sh/uv/getting-started/installation/)
  - Benefits: 10-100x faster dependency resolution, deterministic builds

### Version Control

- **Git**: Version 2.20 or higher (for development/contribution)

## Coverage Provider Requirements

test-coverage-mcp requires at least one coverage provider to be configured:

### Codecov Provider

- **API Token**: Required for authentication
- **Repository**: Must be registered with Codecov
- **Permissions**: API token must have read access to coverage data

### Coveralls Provider

- **API Token**: Required for authentication
- **Repository**: Must be registered with Coveralls
- **Permissions**: API token must have read access to coverage data

See [Configuration Guide](../guides/configuration.md) for provider setup instructions.

## Development Tools (Included)

These tools are automatically configured when you install test-coverage-mcp:

- **Testing**: pytest, pytest-cov, pytest-asyncio
- **Code Quality**: ruff (linting), mypy (type checking), black (formatting)
- **Pre-commit Hooks**: Automated code quality checks
- **CI/CD**: GitHub Actions workflows

## Optional Requirements

### For Documentation Development

- **Node.js**: Version 16+ (for Docusaurus documentation)
- **npm/pnpm**: For managing documentation dependencies

### For Docker Deployment

- **Docker**: Version 20.10.0 or later
- **Docker Compose**: Version 2.0+ (for multi-service applications)

### For Development/Contributing

- **IDE Extensions**: Python, Ruff, MyPy support (VS Code, PyCharm recommended)
- **Terminal**: Modern terminal with color support

## Verification Commands

After installing prerequisites, verify your setup:

```bash
# Check Python version
python --version

# Check pip installation
pip --version

# Check Git (if contributing)
git --version

# Verify test-coverage-mcp installation
python -c "import test_coverage_mcp; print(test_coverage_mcp.__version__)"
```

All commands should return version numbers without errors to proceed with test-coverage-mcp setup.
